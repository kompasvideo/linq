Introduction

Although it is not a very common scenario, it is sometimes useful to have the current record change when an item is selected in a bound combobox.

Description

Many programmers expect this to work right out of the box, but it is not as simple as that.

The normal scenario is that a combobox is filled with a list of value-description pairs, by setting the DataSource, ValueMember and DisplayMember to the right values. 

The ComboBox selectedvalue property is then bound to another datasource, which only needs the underlying ValueMember data of the current item in the ComboBox as a foreign key.

The scenario here is that other controls are bound to the same DataSource that is used to fill the ComboBox,

and we expect the current record to change in all those other bound controls, when we change the selection in the ComboBox.

When binding is used, there is always a BindingContext. You can consider this as a kind of collection where all the binding stuff between datasources and controls are kept. 

By default, the BindingContext of the Form is used. It's a kind of dependency property; if there's no instance created for the control itself, it goes looking for the BindingContext of its parent, the Form.

Within that BindingContext, a collection of BindingManagerBase objects are kept. These objects keep track of the current record of a certain datasource that is used in a binding. They are created automatically for any combination of DataSource and DataMember values that are mentioned in a DataBinding creation. 

You can compare a BindingManagerBase somehow with a BindingNavigator:

It keeps track of the current record, and it has ways to change the current record.

By setting the Position property, one can change the current record:

Dim cm As BindingManagerBase =  _ 
CType(ComboBox1.BindingContext(ComboBox1.DataSource, ComboBox1.ValueMember), BindingManagerBase)
cm.Position = ComboBox1.SelectedIndex
 
Dim cm As BindingManagerBase =  _  
CType(ComboBox1.BindingContext(ComboBox1.DataSource, ComboBox1.ValueMember), BindingManagerBase) 
cm.Position = ComboBox1.SelectedIndex 
 

As you can see, the bindingcontext needs 2 keys to get the bindingmanager; datasource and datamember.

There are different ways that you can use datasource and datamember, but in order to get the right bindingmanager, one needs to use them in a consistent way.
