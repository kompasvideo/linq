В образце программы SimpleVariance демонстрируется реализованная в C# 4.0 поддержка ковариантности и контравариантности при использовании универсальных типов в делегатах и интерфейсах. В C# 3.0 универсальные типы были инвариантными. В связи с этим данный образец компилируется без ошибок в C# 4.0, однако не компилируется в предыдущих версиях C#.

Сначала в программе объявляются два простых класса и два делегата:

class  Animal { }
class Cat : Animal { }

delegate T Func1<out T>();
delegate void Action1<in T>(T a);

Затем в ней реализуются делегаты и используются классы:

Func1<Cat> cat = () => new Cat();
Action1<Animal> act1 = (ani) => { Console.WriteLine(ani); };

После этого в программе выполняются присваивания, для которых требуется поддержка ковариантности и контравариантности:

Func1<Animal> animal = cat;
Action1<Cat> cat1 = act1;

Эти присваивания поддерживаются в C# 4.0 и не поддерживаются в предыдущих версиях C#. Первое присваивание демонстрирует ковариантность, а второе — контравариантность.

С помощью новых контекстных ключевых слов out и in можно задать передачу универсального типа в метод делегата или интерфейса, либо его возврат из метода делегата или интерфейса.
